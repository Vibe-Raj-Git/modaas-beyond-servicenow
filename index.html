01
Architectural Consistency - Own Platform, Own Stack
MoDaaS is a sovereign AI platform. Using ServiceNow - a proprietary third-party SaaS - as the order management layer was architecturally contradictory. You cannot credibly sell sovereignty while your own order pipeline depends on an external vendor's infrastructure.

Replacing ServiceNow with Cognizant Neuro SAN means the entire platform - from intent capture to governance to inference delivery - runs on a stack we own, understand, and can audit completely.
Architectural Integrity
02
Zero Third-Party Licence Dependency
ServiceNow licences are priced per user and per module. For a TMF Catalyst project built to demonstrate scalable sovereign AI connectivity across five champion telcos, carrying a ServiceNow dependency was a commercial liability - both for the demo and for eventual productisation.

Neuro SAN is open-source (Apache 2.0). The only infrastructure cost is compute. That is a fundamentally better commercial model for an operator-sold managed service.
Commercial Model
03
Natural Language Replaces Rigid Forms
ServiceNow's service catalog works through structured forms - every field explicitly presented, every value entered in a predefined sequence. For enterprise AI inference ordering with 20+ parameters, this is operationally slow and UX-hostile.

Neuro SAN's Customer Agent extracts all fields from a single natural language input - or a voice one-liner spoken in 60 seconds. No form redesign when requirements change. No script maintenance. No rigid sequence.
UX Superiority
04
TMF921 Native Output - No Transformation Layer
ServiceNow outputs a generic ticket or service request record. To comply with TMF921 Intent Management API v1.1.0, a transformation layer would be needed to map ServiceNow's data model to the TMF schema.

Neuro SAN's tmf921_generator CodedTool a fully compliant TMF921 intent document directly - including the approved LLM model list, a tamper-evident document fingerprint, service period in international date standard format, and all customer and network parameters structured per the TMF schema. No transformation. No mapping. Standards-native from the start.
Standards Compliance
05
Multi-Agent Orchestration Built In
ServiceNow orchestrates workflows via Flow Designer - a point-and-click automation tool. It has no concept of autonomous agents reasoning about intent, validating governance rules, or making LLM-driven decisions.

Neuro SAN is designed for multi-agent orchestration. The Customer Agent and NAI Governance Agent are Neuro SAN agent networks running as independent processes - reasoning, delegating, and completing tasks autonomously. This is not a workflow - it is an agentic system.
Agentic Architecture
06
Voice, Audio, and Multi-Intent - Capabilities ServiceNow Never Had
It has a complete voice interface - continuous speech recognition, 2.5-second silence gate, barge-in interruption detection, and multi-intent chaining across sessions. A customer can submit three orders across three corridors entirely by voice.

ServiceNow's Virtual Agent supports basic chat. It has no native voice interface, no audio synthesis, no barge-in capability.
New Capabilities

Section 02

What MoDaaS Fully Replaced

Strictly within the MoDaaS project scope - these are the ServiceNow capabilities that Neuro SAN has fully replaced in production. Not theoretically. Actually replaced, tested, and running.

Service Catalog UI
✓ Full
ServiceNow's catalog presented 20 structured form fields. Neuro SAN's Customer Agent accepts a single free-text or spoken sentence and extracts all fields in one pass - dynamically, batch, with no rigid sequence. The agent only asks for genuinely missing values.
Request Management
✓ Full
ServiceNow managed the request lifecycle from submission to BSS acknowledgement. In MoDaaS, the bss_submitter CodedTool submits directly to Etiya BSS via HTTP POST and receives BSS-REF acknowledgement - no ServiceNow intermediary.
Intent Document Generation
✓ Full
ServiceNow would have required a custom transformation layer to produce TMF921-compliant documents. The tmf921_generator CodedTool produces a fully compliant TMF921 intent.
Virtual Agent / Chatbot
✓ Full
ServiceNow's Virtual Agent uses decision-tree NLU with rule-based intent matching. Neuro SAN's front-man agent uses Azure AI for full natural language understanding - no intent taxonomy, no training data, no scripting required.
Location Validation
✓ Full
ServiceNow would require a custom integration or business rule for sovereignty zone validation. The location_validator CodedTool deterministically validates source and target locations against GCC and EU zone lists - no LLM uncertainty in the governance path.
Governance / Compliance Rules
✓ Full
The NAI Governance Agent applies all five TMF915 rules (sovereignty zone, green score, confidentiality, latency, token prices) as deterministic Rule Engines - no LLM in the decision path. ServiceNow's GRC would have required custom scripting and would still lack TMF915 awareness.
Order Status Tracking
✓ Full
The Orders page polls the Backend & Fulfillment directly to provide live visibility. It tracks every request through the Acknowledged, Planning, and Active stages, displaying the order history and network path details via an pipeline visualization. ServiceNow's order status views are replaced entirely by the MoDaaS portal.
Multi-Session Intent Chaining
✓ Full + New
After each BSS submission, the agent offers to chain another intent in the same session - voice or text. A customer can submit three orders for Colt, STC, and Turk Telekom in sequence without resetting. ServiceNow had no equivalent capability.

Technical Depth - Why This Holds in Production
A
sly_data - Sovereign Credential Channel
ServiceNow passes credentials through workflow fields - visible to the platform, logged, and potentially exposed. Neuro SAN's sly_data is a secure side channel: sensitive values like the sovereigntyToken flow between agents and CodedTools without ever entering the LLM chat stream or conversation history.

In MoDaaS, this means the SHA256 sovereignty proof token - generated by NAI and forwarded via OSS → TC → LLM Router - is never visible to the language model at any point. ServiceNow has no equivalent native mechanism.
Security by Architecture
B
chat_context - Native Multi-Turn Memory
ServiceNow's Virtual Agent requires custom state management code to remember values across conversation turns. Neuro SAN natively manages multi-turn context via the chat_context object - nested inside the response payload, correctly persisted across every turn.

In MoDaaS, a customer can provide all 20+ intent fields across multiple messages - or one spoken sentence - and the agent accumulates, remembers, and uses all of them correctly. No custom state code. No session storage. Native to the framework.
Stateful Conversation
C
_to_float() - LLM-Agnostic Data Quality
LLMs return verbose numeric strings: "less than 9ms", "approximately 3 dollars", "nine milliseconds". If a governance rule does a numeric comparison on these strings, it silently fails or throws.

The _to_float() helper uses regex to extract the actual float from any LLM output format. This means RULE-004 (latency < 10ms) and RULE-005 (token prices) work correctly regardless of which LLM is deployed. ServiceNow's scripted business rules would need explicit format enforcement - fragile by comparison.
LLM-Agnostic Robustness
D
Adaptive HOCON Extraction - Zero Script Maintenance
Phase 1 required scripted, sequential question flows - every field asked explicitly in order. Phase 2 switched to adaptive batch extraction via a single HOCON instruction change: the agent extracts all fields present in any single message in one pass, asks only for genuinely missing values, and handles spoken formats (audio mode) and text formats identically.

When the Telefónica scenario added EUR pricing, the change was one line in the HOCON - not a form redesign. ServiceNow's catalog cannot do this without a release cycle.
Zero Maintenance Forms

Section 04

The Honest Pattern

After scanning the entire ServiceNow portfolio, a clear and consistent boundary emerges. It is not about capability - it is about the nature of the problem.

Neuro SAN fully or strongly replaces every ServiceNow module where the core function is conversational intake, intelligent routing, or rule-based decision orchestration.

It cannot replace modules where the core function is structured data modelling, physical discovery, financial planning, or deterministic scheduling - those are not reasoning problems.

✓ Neuro SAN Wins Here

  • Conversational intake of any complexity - text or voice
  • Dynamic field extraction without scripted forms
  • Rule-based governance decisions in deterministic Python
  • Multi-agent orchestration across vendor systems
  • LLM-powered knowledge retrieval via RAG
  • Intent routing based on semantic understanding
  • Autonomous workflow execution without human steps
  • Voice-first interface - orders spoken in 60 seconds
  • TMF-standard document generation (TMF921, TMF915)
  • Multi-intent chaining across sessions
  • Security-by-default via sly_data private channel
  • LLM-agnostic - swap Azure AI model for any model, zero code change

✗ ServiceNow Holds Ground Here

  • Network and hardware discovery (CMDB population)
  • IT asset lifecycle and licence tracking at scale
  • Strategic portfolio and financial planning models
  • Multi-variable constraint scheduling (field technicians)
  • 200+ certified, maintained integration connectors
  • Gantt / project timeline visualisation and management
  • Enterprise compliance certification (SOC2, ISO27001)
  • On-call routing and escalation paging engines
  • HRIS deep integration (Workday, SAP SuccessFactors)
  • Mobile offline field service applications
  • CAB change advisory board workflow models
  • Low-code application development by non-developers

The commercial implication is direct: Neuro SAN eliminates ServiceNow from every customer-facing and intelligence-driven workflow in a telco BSS/OSS stack - the order capture layer, the governance layer, the AI inference layer - while ServiceNow retains relevance only in back-office structured data management that was never its competitive differentiator anyway.
